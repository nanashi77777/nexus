# NEXUS知因 - 企业级智能知识管理平台技术方案

## 1. 项目概述

### 1.1 项目背景
NEXUS知因是一个基于现代化微服务架构构建的企业级智能知识管理平台。旨在通过先进的AI技术，解决企业/个人在知识沉淀、检索与应用过程中的痛点，提供从文档解析、知识库构建到智能问答的全链路解决方案。

### 1.2 建设目标
- **构建高可用基础设施**：基于Spring Boot 3与Java 21，打造稳定、高性能的服务端基座。
- **实现智能化知识处理**：集成LangChain4j与大模型能力，实现文档自动解析、向量化与语义检索。
- **保障数据安全与权限**：建立完善的认证授权体系与数据隔离机制。
- **提供灵活的扩展能力**：采用六边形架构，确保业务逻辑与基础设施解耦，便于技术迭代。

---

## 2. 总体架构设计

### 2.1 架构风格
项目采用 **六边形架构 (Hexagonal Architecture)**，亦称端口与适配器架构。核心思想是将业务逻辑作为系统的核心，通过端口（Ports）与外部世界（Adapters）进行交互，实现业务与技术的彻底解耦。

### 2.2 分层架构
系统自内向外划分为以下四层：

1.  **Application Layer (应用层)**
    - **职责**：包含核心业务逻辑、用例编排与领域模型。
    - **特征**：纯净的Java代码，不依赖任何具体技术框架（如Spring、MyBatis），仅通过接口（Port）定义所需能力。
    - **核心组件**：Service (业务流程), Port (接口定义)。

2.  **Infrastructure Layer (基础设施层)**
    - **职责**：提供应用层所需的技术实现。
    - **特征**：包含数据库访问、缓存、消息队列、AI模型调用等具体实现。
    - **核心组件**：Adapter (适配器实现), RepositoryImpl (持久化实现)。

3.  **Web Layer (表现层)**
    - **职责**：处理HTTP请求，适配前端交互。
    - **特征**：RESTful API设计，负责参数校验与结果封装。
    - **核心组件**：Controller, DTO (数据传输对象)。

4.  **Common Layer (公共层)**
    - **职责**：提供跨层通用的工具、常量与模型。
    - **核心组件**：GlobalException, ResultVO, Utils。

## 3. 技术栈选型

### 3.1 后端核心
- **开发语言**：Java 21 (LTS)
- **核心框架**：Spring Boot 3.5.0
- **构建工具**：Gradle (Kotlin DSL)

### 3.2 数据存储
- **关系型数据库**：PostgreSQL 12+ (支持复杂查询与JSONB)
- **缓存/会话**：Redis 6+ (高性能缓存与分布式锁)
- **向量存储**：PGVector (基于PostgreSQL的向量扩展，支持高效相似度搜索)
- **ORM框架**：Xbatis (增强版MyBatis，灵活高效)
- **连接池**：Druid

### 3.3 AI与智能化
- **AI应用框架**：LangChain4j (统一的大模型交互接口)
- **文档解析**：Apache Tika (支持PDF, Word, HTML等多格式解析)
- **Token计算**：JTokkit
- **向量化模型**：集成OpenAI/DashScope等Embedding模型

### 3.4 安全与认证
- **认证框架**：Sa-Token 1.44.0 (轻量级Java权限认证框架)
- **会话管理**：Redis集成 (分布式Session)
- **加密算法**：BCrypt (密码加密), AES (敏感数据加密)

### 3.5 基础设施与工具
- **分布式锁**：Redisson
- **布隆过滤器**：Redis Bloom Filter (防止缓存穿透)
- **工具库**：Hutool 5.8, Lombok
- **API文档**：SpringDoc OpenAPI (Swagger 3)
- **异步任务**：自定义Task Executor框架 (支持长/短任务分离)

## 4. 核心功能模块设计

### 4.1 用户与权限模块
- **功能**：用户注册、登录、状态管理（激活/禁用）、邀请码机制。
- **设计**：
    - 使用雪花算法生成全局唯一User ID。
    - 密码采用加盐哈希存储。
    - 基于Sa-Token实现RBAC权限控制与StpUtil会话管理。

### 4.2 资源管理模块 (RAG核心)
- **功能**：多格式文档上传、链接导入、AI生成内容存储。
- **流程**：
    1.  **上传/导入**：记录资源元数据 (Resources Table)。
    2.  **解析 (Parsing)**：使用Apache Tika提取文本内容。
    3.  **分片 (Chunking)**：基于语义或字符数将文本切分为Resource Chunks。
    4.  **向量化 (Embedding)**：调用Embedding Model将文本转为向量。
    5.  **存储**：将向量与文本存入PGVector，构建索引。

### 4.3 学习空间 (Isolation)
- **功能**：提供逻辑隔离的知识库环境。
- **设计**：
    - 资源与分片均关联至特定的 `learning_space_id`。
    - 检索时限定在当前空间范围内，保障数据隐私与相关性。

### 4.4 异步任务系统
- **背景**：文档解析与向量化为耗时操作。
- **设计**：
    - **任务表**：持久化任务状态、参数与执行结果。
    - **调度器**：区分 `Long-Running` (AI生成、解析) 与 `Short-Running` 任务。
    - **容错**：支持任务重试与失败记录。

## 5. 数据库设计概览

### 5.1 用户域 (User Domain)
- **`user_accounts`**: 存储用户基础信息、密码、状态、邀请关系。
    

### 5.2 资源域 (Resource Domain)
- **`resources`**: 存储原始文件/链接的元数据。
    
- **`resource_chunks`**: 存储切分后的文本块及其向量引用。
    

### 5.3 学习空间域 (Learning Space Domain)
- **`learning_spaces`**: 学习空间表，实现多租户数据隔离的核心。
   

### 5.4 智能体域 (Agent Domain)
- **`agent_chat_sessions`**: 存储用户与AI的会话状态。
   
- **`agent_chat_messages`**: 存储会话中的具体消息记录。
   

### 5.5 知识域 (Knowledge Domain)
- **`knowledge_points`**: 知识点元数据表。
    
- **`knowledge_point_versions`**: 知识点版本控制表，存储具体的知识内容。
    
- **`knowledge_graphs`**: 知识图谱元数据表。
   
- **`graph_nodes`**: 图谱节点表，关联具体的知识点。
   
- **`graph_edges`**: 图谱边表，定义节点间的关系。
   

### 5.6 讲解域 (Explanation Domain)
- **`explanation_documents`**: 讲解文档表，用于生成结构化的知识讲解。

## 6. API 设计规范

遵循 RESTful 风格，统一响应格式：

- **URL路径**：`/api/v1/{resource}`
- **HTTP方法**：GET (查询), POST (创建/复杂查询), PUT (全量更新), PATCH (部分更新), DELETE (删除)。
- **统一响应结构 (`ResultVO`)**：
  ```json
  {
    "code": 200,
    "message": "Success",
    "data": { ... },
    "timestamp": 1715678901234
  }
  ```
- **异常处理**：全局异常处理器捕获 `ApplicationException` (业务异常) 与 `InfrastructureException` (技术异常)，映射为标准HTTP状态码与错误信息。

## 7. 部署与运维

- **容器化**：提供 `Dockerfile`，支持一键构建应用镜像。
- **配置管理**：
    - `application.yml`: 通用配置。
    - `application-{profile}.yml`: 环境差异化配置 (dev/prod)。
    - 敏感信息 (DB密码, API Key) 通过环境变量注入。
- **CI/CD**：基于Gradle构建，支持自动化测试与打包。

## 8. 总结
本方案基于成熟的Spring Boot生态与前沿的AI技术栈，构建了一个高内聚、低耦合、易扩展的企业级知识管理平台。六边形架构保证了系统的长期可维护性，而LangChain4j与PGVector的结合则为系统赋予了强大的智能化核心能力。
